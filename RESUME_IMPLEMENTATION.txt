🚀 COMMENT UTILISER
════════════════════════════════════════════════════════════════════════════

ÉTAPE 1: Exécuter la migration
   $ php bin/console doctrine:migrations:migrate

ÉTAPE 2: Démarrer le serveur
   $ symfony server:start

ÉTAPE 3: Accéder aux formulaires

   FILIÈRES:
   • Créer: http://localhost:8000/fil/new
   • Lister: http://localhost:8000/fil
   • Éditer: http://localhost:8000/fil/{id}/edit

   ÉTABLISSEMENTS:
   • Créer: http://localhost:8000/eta/new
   • Lister: http://localhost:8000/eta
   • Éditer: http://localhost:8000/eta/{id}/edit

   UTILISATEURS:
   • Créer: http://localhost:8000/utilisateur/new
   • Lister: http://localhost:8000/utilisateur
   • Éditer: http://localhost:8000/utilisateur/{id}/edit

ÉTAPE 4: Ajouter des images

   1. Allez sur Google Images (https://www.google.com/imghp)
   2. Trouvez une image
   3. Cliquez droit → "Copier l'adresse de l'image"
   4. Collez dans le champ "URL Image (Google Images)"
   5. L'aperçu s'affiche en temps réel ✓
   6. Cliquez "Enregistrer" ✓

ÉTAPE 5: Voir le résultat

   • Pages affichent maintenant 4 IMAGES PAR LIGNE (responsive)
   • Hauteur fixe: 200px
   • Effet hover: zoom + ombre
   • Placeholder pour images manquantes

❓ DÉPANNAGE
════════════════════════════════════════════════════════════════════════════

PROBLÈME: "Route not found"
→ Vérifiez: http://localhost:8000/ (port 8000)

PROBLÈME: Image n'affiche pas
→ L'URL peut être expirée (Google Images liens temporaires)
→ Utilisez une autre image depuis Unsplash ou Pexels

PROBLÈME: Migration échoue
→ Vérifiez votre BD MySQL/PostgreSQL est running
→ Vérifiez .env: DATABASE_URL est correct

PROBLÈME: Formulaire ne sauvegarde pas
→ Remplissez tous les champs obligatoires
→ Vérifiez les erreurs de validation affichées
